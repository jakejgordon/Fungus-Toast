# FungusToast Analytics

This folder contains offline balance analysis tooling for simulation exports.

## Preferred Workflow

When collaborating with GitHub Copilot in this repository, the preferred default is:

- If the user asks to run simulations, Copilot runs them directly.
- If the user asks to run analytics, Copilot runs them directly.
- Only fall back to manual user-run steps when terminal/tool execution is blocked.

## Inputs

Point the script at a simulation Parquet export folder generated by `FungusToast.Simulation`.
Expected files:

- `games.parquet`
- `players.parquet`
- `mutations.parquet`
- `mycovariants.parquet`
- `upgrade_events.parquet`
- `manifest.json`

Default export root is:

- `FungusToast.Simulation/bin/Debug/net8.0/SimulationParquet/<experiment-id>`

## Setup

```powershell
cd FungusToast.Analytics
python -m pip install -r requirements.txt
```

## Usage

```powershell
python analyze_balance.py --run-folder "..\FungusToast.Simulation\bin\Debug\net8.0\SimulationParquet\exp_20260306T170000"
```

Optional output directory:

```powershell
python analyze_balance.py --run-folder "<path>" --output-dir "<path>"
```

## Outputs

- `mutation_recommendations.csv`
- `mycovariant_recommendations.csv`
- `balance_recommendations.md`

The markdown report includes ranked OP/UP candidates and confidence-aware diagnostics.

## Upgrade Event Logging

`upgrade_events.parquet` contains ordered mutation upgrade events per player/game.
Useful fields include:

- `Round`, `MutationName`, `MutationTier`
- `OldLevel`, `NewLevel`
- `MutationPointsBefore`, `MutationPointsAfter`, `PointsSpent`
- `UpgradeSource` (`manual`, `surge`, `auto`)

This enables build-order and timing analyses beyond end-state mutation levels.
